 <span data-ttu-id="46baa-110">Svcutil. exe, hizmetten alınan Web Hizmetleri Açıklama Dili (WSDL) veya ilke dosyasını temel alarak istemciyi oluşturur.</span><span class="sxs-lookup"><span data-stu-id="46baa-110">Svcutil.exe generates the client based on the Web Services Description Language (WSDL) or policy file received from the service.</span></span> <span data-ttu-id="46baa-111">Kullanıcı asıl adı (UPN), Kullanıcı adının "\@" ile bitiştirerek ve tam nitelikli etki alanı adı (FQDN) eklenerek oluşturulur.</span><span class="sxs-lookup"><span data-stu-id="46baa-111">The user principal name (UPN) is generated by concatenating the user name with "\@" and then adding a fully-qualified domain name (FQDN).</span></span> <span data-ttu-id="46baa-112">Ancak, Active Directory kayıtlı kullanıcılar için bu biçim geçerli değildir ve aracın ürettiği UPN, Kerberos kimlik doğrulamasında şu hata iletisiyle bir hataya neden olur: **Oturum açma girişimi başarısız oldu.**</span><span class="sxs-lookup"><span data-stu-id="46baa-112">However, for users who registered on Active Directory, this format is not valid and the UPN that the tool generates causes a failure in the Kerberos authentication with the following error message: **The logon attempt failed.**</span></span> <span data-ttu-id="46baa-113">Bu sorunu çözmek için, aracın ürettiği istemci dosyasını el ile onarın.</span><span class="sxs-lookup"><span data-stu-id="46baa-113">To resolve this problem, manually fix the client file that the tool generated.</span></span>

