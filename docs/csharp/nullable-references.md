## <a name="nullable-annotation-context"></a><span data-ttu-id="f76bb-182">Null yapılabilir ek açıklama bağlamı</span><span class="sxs-lookup"><span data-stu-id="f76bb-182">Nullable annotation context</span></span>

<span data-ttu-id="f76bb-183">Derleyici, devre dışı bırakılmış bir null yapılabilir ek açıklama bağlamında aşağıdaki kuralları kullanır:</span><span class="sxs-lookup"><span data-stu-id="f76bb-183">The compiler uses the following rules in a disabled nullable annotation context:</span></span>

<span data-ttu-id="f76bb-189">Derleyici, etkinleştirilmiş bir null yapılabilir ek açıklama bağlamında aşağıdaki kuralları kullanır:</span><span class="sxs-lookup"><span data-stu-id="f76bb-189">The compiler uses the following rules in an enabled nullable annotation context:</span></span>

- <span data-ttu-id="f76bb-190">Başvuru türündeki herhangi bir değişken **null atanamaz bir başvurudur**.</span><span class="sxs-lookup"><span data-stu-id="f76bb-190">Any variable of a reference type is a **non-nullable reference**.</span></span>
- <span data-ttu-id="f76bb-191">Null olamayan herhangi bir başvuruya, güvenli bir şekilde başvurulmalıdır.</span><span class="sxs-lookup"><span data-stu-id="f76bb-191">Any non-nullable reference may be dereferenced safely.</span></span>

- <span data-ttu-id="f76bb-195">Null yapılabilir bir başvurunun null olmadığını bildirmek için null-forverme işlecini kullanabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="f76bb-195">You can use the null-forgiving operator to declare that a nullable reference isn't null.</span></span>

<span data-ttu-id="f76bb-196">Etkin bir null yapılabilir ek açıklama bağlamında, bir başvuru türüne eklenen `?` karakteri **null olabilen bir başvuru türü**bildirir.</span><span class="sxs-lookup"><span data-stu-id="f76bb-196">In an enabled nullable annotation context, the `?` character appended to a reference type declares a **nullable reference type**.</span></span> <span data-ttu-id="f76bb-197">İfadenin null olmadığını bildirmek için `!` **null-forverme işleci** bir ifadeye eklenebilir.</span><span class="sxs-lookup"><span data-stu-id="f76bb-197">The **null-forgiving operator** `!` may be appended to an expression to declare that the expression isn't null.</span></span>

## <a name="nullable-warning-context"></a><span data-ttu-id="f76bb-198">Null yapılabilir uyarı bağlamı</span><span class="sxs-lookup"><span data-stu-id="f76bb-198">Nullable warning context</span></span>

1. <span data-ttu-id="f76bb-205">Değişken, null olmayan bir değere kesin olarak atandı.</span><span class="sxs-lookup"><span data-stu-id="f76bb-205">The variable has been definitely assigned to a non-null value.</span></span>
1. <span data-ttu-id="f76bb-206">Değişken veya ifade, kendisine başvurulmadan önce null değere karşı denetlendi.</span><span class="sxs-lookup"><span data-stu-id="f76bb-206">The variable or expression has been checked against null before de-referencing it.</span></span>

<span data-ttu-id="f76bb-207">Null yapılabilir uyarı bağlamı `enabled` olduğunda, derleyici bir değişken veya ifadeye, **belki de boş** bir durumda her başvurduğunda uyarı oluşturur.</span><span class="sxs-lookup"><span data-stu-id="f76bb-207">The compiler generates warnings whenever you dereference a variable or expression in a **maybe null** state when the nullable warning context is `enabled`.</span></span> <span data-ttu-id="f76bb-208">Ayrıca, null olabilen ek açıklama bağlamı `enabled` olduğunda, null olmayan bir başvuruya veya ifadeye **null** değer atanabilir bir başvuru türüne atandığında uyarılar oluşturulur.</span><span class="sxs-lookup"><span data-stu-id="f76bb-208">Furthermore, warnings are generated when a **maybe null** variable or expression is assigned to a nonnullable reference type when the nullable annotation context is `enabled`.</span></span>
