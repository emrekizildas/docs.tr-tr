## <a name="nullable-annotation-context"></a><span data-ttu-id="2f523-183">Null yapılabilir ek açıklama bağlamı</span><span class="sxs-lookup"><span data-stu-id="2f523-183">Nullable annotation context</span></span>

<span data-ttu-id="2f523-184">Derleyici, devre dışı bırakılmış bir null yapılabilir ek açıklama bağlamında aşağıdaki kuralları kullanır:</span><span class="sxs-lookup"><span data-stu-id="2f523-184">The compiler uses the following rules in a disabled nullable annotation context:</span></span>

<span data-ttu-id="2f523-190">Derleyici, etkinleştirilmiş bir null yapılabilir ek açıklama bağlamında aşağıdaki kuralları kullanır:</span><span class="sxs-lookup"><span data-stu-id="2f523-190">The compiler uses the following rules in an enabled nullable annotation context:</span></span>

- <span data-ttu-id="2f523-191">Başvuru türündeki herhangi bir değişken **null atanamaz bir başvurudur**.</span><span class="sxs-lookup"><span data-stu-id="2f523-191">Any variable of a reference type is a **non-nullable reference**.</span></span>

- <span data-ttu-id="2f523-193">Herhangi bir Nullable başvuru türü (değişken `?` bildiriminde bulunan tür öğesinden sonra belirtilen) null olabilir.</span><span class="sxs-lookup"><span data-stu-id="2f523-193">Any nullable reference type (noted by `?` after the type in the variable declaration) may be null.</span></span> <span data-ttu-id="2f523-194">Statik analiz, başvurunun başvurulduğunu null dışında bir değer olarak bilinmeyeceğini belirler.</span><span class="sxs-lookup"><span data-stu-id="2f523-194">Static analysis determines if the value is known to be non-null when it is dereferenced.</span></span> <span data-ttu-id="2f523-195">Aksi takdirde, derleyici sizi uyarır.</span><span class="sxs-lookup"><span data-stu-id="2f523-195">If not, the compiler warns you.</span></span>

<span data-ttu-id="2f523-197">Etkin bir null yapılabilir ek açıklama bağlamında, `?` başvuru türüne eklenen karakter **null yapılabilir bir başvuru türü**bildirir.</span><span class="sxs-lookup"><span data-stu-id="2f523-197">In an enabled nullable annotation context, the `?` character appended to a reference type declares a **nullable reference type**.</span></span> <span data-ttu-id="2f523-198">İfadenin null olmadığını bildirmek için **null forgilelik işleci** (`!`) bir ifadeye eklenebilir.</span><span class="sxs-lookup"><span data-stu-id="2f523-198">The **null forgiveness operator** (`!`) may be appended to an expression to declare that the expression isn't null.</span></span>

1. <span data-ttu-id="2f523-206">Değişken, null olmayan bir değere kesin olarak atandı.</span><span class="sxs-lookup"><span data-stu-id="2f523-206">The variable has been definitely assigned to a non-null value.</span></span>
1. <span data-ttu-id="2f523-207">Değişken veya ifade, kendisine başvurulmadan önce null değere karşı denetlendi.</span><span class="sxs-lookup"><span data-stu-id="2f523-207">The variable or expression has been checked against null before de-referencing it.</span></span>

<span data-ttu-id="2f523-208">Derleyici, **null** yapılabilir uyarı bağlamı `enabled`olduğunda bir değişkene veya ifadeye, belki de null bir durumda her başvuru yaptığınızda uyarı oluşturur.</span><span class="sxs-lookup"><span data-stu-id="2f523-208">The compiler generates warnings whenever you dereference a variable or expression in a **maybe null** state when the nullable warning context is `enabled`.</span></span> <span data-ttu-id="2f523-209">Ayrıca, **null** olabilen ek açıklama bağlamı `enabled`olduğunda, null olamayan bir başvuruya veya ifadeye null olmayan bir başvuru türüne atandığında uyarılar üretilir.</span><span class="sxs-lookup"><span data-stu-id="2f523-209">Furthermore, warnings are generated when a **maybe null** variable or expression is assigned to a nonnullable reference type when the nullable annotation context is `enabled`.</span></span>
